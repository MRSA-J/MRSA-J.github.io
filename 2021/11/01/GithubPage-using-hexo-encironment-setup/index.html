Build GithubPage blog

Hexo is a blog framework.It generalize local files to a wesite. If you don’t want to generalize a website from local files, you don’t need Github Pages (Of course, you can use it to store your portfolio and anything you want).

Before using Hexo,you need to have Node.js and Git installed.

Attention: You can use nvm to control the version of npm

After installing Node.js and Git, start to install Hexo

Hexo Installation

  • Open cmd
  • Type npm install -g hexo-clior npm install -g hexo Enter.
  • Typehexo -v to check if install succesfully(get hexo-cli:xxx if succesfully installed)

Build local blog

  • Create a folder to store blog
  • Right click on the folder :Git Bash Here
  • In git Bash, type:
  • hexo init
  • npm install which helps install dependencies
  • hexo ggenerate website
  • hexo srun server/wesite
  • can use http://localhost:4000/ to visit and check the work we have done

Create a new blog and released it

  • Type in gitbash: hexo new "newly released blog name"
  • Edit blogname.md generated in the folder ‘source_posts’
  • hexo g+hexo s,then can see what we have done from local host: http://localhost:4000/

Next theme

There are many theme in Hexo, you can build your own if you wanted.
Here, we use theme: Next[^1] for this post and also for my blog because I think it is relatively concise

Next theme has Chinese doc because it is built by Chinese.

Install Next theme

  1. Git Bash: git clone https://github.com/iissnan/hexo-theme-next themes/next(In the newly version, change the command line to git clone https://github.com/theme-next/hexo-theme-next.git themes/next)

  2. Change blog’s config file:

    Functional config:

    • open blog config file_config.yml
    • find theme:
    • change Hexo’s theme from default theme lanscape to next theme: next

    Other config:

    • Find # Site, add blog name, author name
    • Choose your language: Setting language to en or zh-Hans (pick English or Chinese)
    • Find # URLand type in your github.io url, i.e.url: https://username.github.io

  3. Regenerate deployment:

    • Git Bash type in hexo g -d (First turn modified content into website, that is generate, then deploy)
